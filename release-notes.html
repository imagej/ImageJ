• 1.51r 29 September 2017
    • Thanks to Norbert Vischer, if the 'type' argument of the Plot.add(type,xpoints,ypoints) macro function and plot.add(type,xpoints,ypoints) method starts with "code: " then the macro code contained in the string is executed per point, using variable names 'x' and 'y' for the point location, 's' for high-resolution scale factor and 'i' for the point index. The Help>Examples>Custom Plot Symbols command provides an example.
    • Thanks to Mathieu Prouveur, ImageJ displays an error message when a macro function attempts to return a string enclosed in parenthesis.
    • Thanks to Roger Leigh, fixed bugs that prevented ImageJ from running on Java 9.
    • Thanks to Derek Xu and Michael Schmid, fixed a bug that could cause the Orthogonal Views command to throw an exception.
    • Thanks to Michael Schmid, fixed a 1.51q reqression that could cause an exception when creating a plot.
